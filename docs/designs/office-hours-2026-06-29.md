## Problem Statement

中文创作者（自媒体、设计师、学生、中小电商运营）需要一个不需要 GPU、不需要翻墙、支持国产 AI 模型的一站式创意设计平台。现有工具存在明显痛点：

- **ComfyUI** — 功能强大但需要本地 GPU、英文界面、学习曲线陡峭
- **Canva/稿定设计** — 易用但模板化、缺乏可编程性、AI 能力有限
- **Midjourney/OpenAI** — 高质量生成但缺乏精细化控制，无法迭代调优
- **现有开源方案**（Franklin Canvas / Node Banana）— 面向海外市场，不支持中文模型和中文 Prompt 优化

用户需要的是：在无限画布上，通过拖拽节点的方式精准可控地完成 AI 图文创作全流程。

## What Makes This Cool

三个核心创新点的交汇：

1. **Excalidraw 手绘美学 × 无限画布** — 不同于所有竞品"严肃工具"的视觉语言，手绘风格让创作过程更轻松、更人性化
2. **节点工作流 × AI 生成** — 把 Blender 几何节点的程序化编辑逻辑带到 AI 创作中，让非技术用户也能精细化控制
3. **多模型适配器 × 中文生态** — 同一套工作流可以无缝切换 OpenAI / 通义 / Stability / 文心等模型，降低厂商锁定风险

## Constraints

- **单人全栈 TypeScript 开发** — 不能太膨胀，每个 Phase 必须 shipping 一件事
- **预算敏感** — AI API 使用 Mock 模式开发调试，仅接入 2-3 家商业 API
- **浏览器端 SPA** — 画布需支持 500+ 元素流畅运行
- **本地存储优先** — MVP 不上云，IndexedDB + Dexie.js
- **MIT 许可证** — Excalidraw fork 必须遵守，商用不受限

## Premises

1. **✅ 中文创作者是核心用户** — 需要支持国产模型、中文 Prompt 优化、无墙访问。
2. **✅ Excalidraw 手绘美学是视觉标识** — 保持并强化，不做成普通严肃工具。
3. **✅ 节点工作流是核心差异化** — 和 Canva 等模板化工具的本质区别。
4. **🟡 视频作为可插拔能力** — 架构上预留视频节点和适配器接口，实际 API 接入放后面。
5. **✅ 本地优先** — IndexedDB + Dexie.js，后端按需引入。

## Approaches Considered

### Approach A: 当前路线图（推荐并已选）
按序推进 8 个 Phase：画布 → 节点编辑器 → 节点引擎 → AI适配器 → AI基础设施 → 后端 → 应用UI → 测试
- **Effort:** L | **Risk:** Low — Phase 1 已验证架构可行性
- **Pros:** 依赖关系清晰，逐层夯实基础设施，减少返工
- **Cons:** Phase 4 才接入 AI，前期缺少"可见价值"

### Approach B: 中文生态优先
把 AI 适配器提前到 Phase 2/3，先做 MockAdapter + 国内 AI 模型接入
- **Effort:** M | **Risk:** Med — 打乱依赖关系，节点后置可能引发重构
- **Pros:** 更快看到"能出图"的效果，利于吸引种子用户
- **Cons:** 没有节点编辑器时 AI 接入的交互方式不明确

## Recommended Approach

**Approach A（当前路线图）**。Phase 1 已完成验证，架构合理。中文生态作为贯穿各 Phase 的底层差异化，在 Phase 4（AI适配器）中重点体现。
