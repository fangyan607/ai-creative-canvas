## Spacing Scale

All values conform to 4px grid (Tailwind v4 default scale).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge padding, inline spacing |
| sm | 8px | Compact element spacing, toggle/switch internal gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, dialog content padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks, page-level vertical rhythm |
| 3xl | 64px | Page-edge margins, large view gaps |

### Exceptions

| Context | Value | Reason |
|---------|-------|--------|
| Sidebar collapsed width | 44px | Icon-only state — matches 44x44px touch target (D-04 user confirmation) |
| Sidebar expanded width | 288px | Fixed-width panel per D-04 |
| Collapsed sidebar icon buttons | 44x44px | Minimum touch target per WCAG 2.2, cramped icon-only layout |
| Sidebar tab icon + label spacing | 8px | Between icon and label text in tabs |

## Typography

Four roles only. All weights: regular (400) and semibold (600).

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 | 1.5 | Primary body text, panel content, description text |
| Label | 12px | 400 | 1.4 | Form labels, tab labels, button text, metadata, timestamps |
| Heading | 20px | 600 | 1.2 | Card titles, section headings, dialog headers |
| Display | 28px | 600 | 1.2 | Page titles (Projects, Settings), empty state headings |

**Source:** Body 14px/1.5 confirmed in App.css line 179-180. Label, Heading, Display roles confirmed by user.

### Notes
- All text uses `--font-sans` (Inter) except monospace/code contexts which use `--font-mono` (JetBrains Mono).
- Button and form label text uses Label role (12px/400/1.4).
- No third font weight (e.g., bold 700) is used. Stick to 400 and 600 only.
- Body line-height 1.5 provides adequate readability at 14px. No exceptions.

## Color

Contract is based on existing CSS custom properties in `App.css` lines 11-50 (light) and 53-81 (dark). Phase 7 uses these definitions unchanged.

### Light Mode

| Role | CSS Variable | Value | Visual | Usage |
|------|-------------|-------|--------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) | #ffffff white | Page backgrounds, modal backdrops, main content area |
| Secondary (30%) | `--secondary` / `--muted` | oklch(0.967 0.002 90) | #f7f7f5 off-white | Cards, sidebar background, navigation, tab list background, panel surfaces |
| Accent (10%) | `--accent` | oklch(0.92 0.04 265.7) | #a29bfe light purple | See reservation list below |

| Muted Foreground | `--muted-foreground` | oklch(0.6 0.02 264.4) | soft gray | Secondary text, placeholder, disabled text |
| Border | `--border` | oklch(0.9 0.005 264.4) | #e6e6e6 hairline | All borders, dividers, card strokes |
| Ring | `--ring` | oklch(0 0 0) | #000000 black | Focus ring, active indicator |

**Source:** Pre-populated from App.css lines 11-50 (Phase 1 design tokens).

### Accent Reservation

Accent (`--accent: #a29bfe` light purple) is reserved **exclusively** for:

| Element | Usage |
|---------|-------|
| Selected tab indicator | Active tab underline in sidebar tab bar |
| Active sidebar item background | Currently selected tab in collapsed sidebar icon nav |
| Toggle switch active state | Fill color when switch is ON |
| Slider range fill | Track segment between the thumb and min |
| Focus ring (secondary elements) | Non-primary focus rings (primary focus ring uses `--ring` black) |
| Link text | Text links in settings, help tooltips |
| Selected project card border | Active/selected card indicator |

**Accent is NOT used for:**
- Primary action buttons (these use black `--primary` background, white text)
- Page headers or section titles (use `--foreground` near-black)
- Navigation indicators outside the sidebar
- Icons (use `--foreground` or `--muted-foreground`)
- Destructive actions (use `--destructive` red)

### Dark Mode

Dark mode CSS variables already defined in App.css lines 53-81. Phase 7 activates dark mode via Tailwind `dark:` class strategy on `<html>` element, with `prefers-color-scheme` system detection.

| Role | Light | Dark | Delta |
|------|-------|------|-------|
| Background | #ffffff | oklch(0.141 0.01 264.4) dark navy | Near-inverted |
| Foreground | #1a1a2e | oklch(0.985 0 0) white | Near-inverted |
| Accent | #a29bfe light purple | oklch(0.37 0.04 265.7) muted purple | Dimmed for dark bg |
| Border | #e6e6e6 | oklch(0.35 0.01 264.4) | Darker hairline |

No additional CSS variable definitions needed. Phase 7 uses existing `.dark` classes.
