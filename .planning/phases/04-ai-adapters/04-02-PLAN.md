    1. Extract `prompt`, `width` (default 1024), `height` (default 1024), `seed` (default random) from nodeData
    2. Emit 'progress' with { percent: 0, stage: 'generating' }
    3. Generate a deterministic hue from prompt string: `hue = [...prompt].reduce((a, c) => a + c.charCodeAt(0), 0) % 360`

       - Line 4 (y=160): `--- MOCK ---`
    7. Font size: `Math.min(width, height) * 0.04px sans-serif`
    8. Emit 'progress' with { percent: 50, stage: 'rendering' }
    9. Convert canvas to Blob via `canvas.convertToBlob({ type: 'image/png' })`
    10. If onStoreImage callback provided, call it with the blob to get imageBlobId
    11. If no onStoreImage callback, generate a mock imageBlobId: `mock-${crypto.randomUUID()}`
    12. Record timing using performance.now()
    13. Emit 'done' with AdapterResult
    14. Return AdapterResult with: imageBlobId, width, height, seed (number), model: 'mock-default', timing (ms)

    - The class constructor accepts an optional `options` object with `{ mode?: 'fallback' | 'manual' }`
    - Fallback mode: returns from execute() immediately without canvas operations (for use during engine initialization when no real API keys are configured)
    - Manual mode: full canvas rendering for demo purposes (default)
    - The mode is stored as `this.mode` and affects behavior minimally — both modes generate images, but 'fallback' mode skips the storeImage callback call and returns a mock imageBlobId directly (faster, no blob storage overhead)

          // This test just verifies the hash function is deterministic
