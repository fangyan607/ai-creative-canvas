## Spacing Scale

Base unit: **8px** (Figma DESIGN.md convention). All values are multiples of 8.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 8px | Compact element spacing, socket gaps, icon padding |
| sm | 12px | PropertyPanel field gap, node internals padding |
| md | 16px | Default element spacing, panel interior padding |
| lg | 24px | Section padding, node body spacing, dialog interior padding |
| xl | 32px | Layout gaps between major columns, modal width inset |
| 2xl | 48px | Major section breaks, color-block section padding |
| section | 96px | Page-level vertical rhythm (deferred to Phase 7) |

**Exceptions:** None declared for this phase.

**Touch targets:** Minimum 44px tap height for all interactive controls in PropertyPanel (applied via `min-h-11` or equivalent padding). Socket handles in React Flow are 12px diameter circles with 24px invisible hit area (React Flow's `handleRadius` + padding).

## Typography

| Role | Size | Weight | Line Height | Used For |
|------|------|--------|-------------|----------|
| Body | 14px | 400 | 1.5 | Node labels, parameter values, panel body text, PropertyPanel content |
| Label | 12px | 500 | 1.4 | PropertyPanel field labels, parameter group headers, tab labels |
| Heading | 16px | 600 | 1.3 | Panel headers ("Layers", "Properties"), node type titles, dialog titles |
| Small | 11px | 400 | 1.3 | Captions, socket labels (hover), metadata, element counts |

**Font stack:** `Inter, system-ui, sans-serif` for body/UI. `JetBrains Mono, SF Mono, monospace` for monospace (code params, metadata).

**Principles:**
- No mid-gray text — hierarchy expressed through weight, not opacity
- Labels always sentence case (not all-caps) unless matching Figma eyebrow pattern
- Socket labels use `Small` (11px) with 0.3px letter-spacing

| Error / Destructive | `#ef4444` | Error states, destructive buttons, invalid connections |

### Accent Reserved For

The accent color (`#000000` black primary) is reserved for:
1. **Primary action buttons** in dialogs ("Confirm", "Apply to Canvas")
2. **Selected state** of template cards (black border/highlight)
3. **Focus mode toggle** active state (Node mode when active)
4. **Output sockets** on all nodes (right-side handles)
5. **Connection wire color** for established edges (black, 2px stroke)

The pastel block colors (`--color-node-*`) are reserved exclusively for node type accent bars — never used for buttons, backgrounds, or text colors.

## Node Visual Specification (D-31 Hybrid Style)

### Node Anatomy

```
┌──────────────────────────────────┐
│ [icon] NodeType Title  [⚙/✕]    │  ← Header bar (12px padding, 16px height)
├──────────────────────────────────┤
│ ● Input                         │  ← Accent bar (4px, pastel per node type)
│                                  │
│  ┌────────────────────────────┐  │
│  │  Parameter area            │  │  ← Body (clean internals, structured layout)
│  │  (fields render per type)  │  │
│  └────────────────────────────┘  │
│                                  │
│                         Output ● │  ← Socket area
└──────────────────────────────────┘
```

**Visual rules:**
- **Node background:** White (`#ffffff`), rounded `--radius-md` (8px)
- **Node border:** 1px `--color-hairline` (`#e6e6e6`)
- **Node shadow (default):** None (flat design per Figma principle)
- **Node shadow (selected):** `0 4px 16px rgba(0,0,0,0.06)`
- **Node shadow (dragging):** `0 8px 24px rgba(0,0,0,0.10)`
- **Accent bar:** 4px tall strip at top of node body, colored per node type
- **Header:** Node type name left-aligned, icon left of name, optional close/minimize right
- **Header text:** `Heading` (16px, 600), spaced `sm` (12px) from accent bar
- **Socket circles:** 12px diameter, 24px invisible hit area, positioned at mid-height on left (inputs) and right (outputs)
- **Socket labels:** Visible on hover or zoom >= 80%, positioned inline next to socket

### Node Dimensions

| Dimension | Value | Notes |
|-----------|-------|-------|
| Min width | 200px | Ensures readable content |
| Max width | 360px | Prevents excessively wide nodes |
| Header height | 16px | Icon + title area (12px vertical padding) |
| Accent bar height | 4px | Colored strip per node type |
| Body padding | 12px interior | Space for parameter fields |
| Socket offset | 24px from edge | React Flow handle position |

## Interaction Specifications

### Drag-Connect (React Flow)

1. **Initiation:** Pointer down on output socket (right side, black)
2. **Preview:** A temporary edge line follows the pointer — green if target is valid, red if invalid
3. **Validation pipeline** (D-35):
   - No self-connections (source !== target)
   - No duplicate connections (same source-target pair)
   - Direction enforced (output socket to input socket only)
   - No cycles (DAG must remain acyclic — checked via `wouldCreateCycle` utility)
4. **Snap:** On valid drop, edge snaps between sockets with a smooth bezier curve
5. **Rejection:** On invalid drop, red preview line disappears with no edge created. No toast/notification for invalid connections — visual rejection is sufficient

### Node Interactions

| Interaction | Behavior |
|-------------|----------|
| Click node | Selects node, open PropertyPanel, shows selection border |
| Drag node | Moves node position (React Flow native), live update in store on release |
| Double-click node | Opens full parameter focus (if PromptNode, focuses textarea) |
| Delete node (keyboard) | Backspace/Delete with confirmation dialog |
| Delete node (header ✕) | Same confirmation dialog |
| Drag new node from toolbar | Creates node at drop position (React Flow drag-and-drop pattern) |

### Focus Mode Toggle (D-30)

| Mode | Canvas Behavior | Node Overlay Behavior | Toggle |
|------|----------------|----------------------|--------|
| Canvas mode | Full Excalidraw interaction (draw, pan, zoom) | Nodes semi-transparent (opacity 40%), non-interactive | Click "Canvas" button or Ctrl+Shift+C |
| Node mode | Canvas interactions disabled (`pointer-events: none` on Excalidraw wrapper) | Nodes fully interactive (select, drag, connect, edit) | Click "Nodes" button or Ctrl+Shift+N |

Toggle is rendered as a segmented button group (2 options) positioned in the top-left corner of the viewport, overlapping both canvas and node overlay. Active mode is filled black (`--color-primary`), inactive is white with black text.
