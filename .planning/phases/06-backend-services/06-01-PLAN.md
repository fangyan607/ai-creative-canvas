<objective>
Build the Hono 4.x backend at `apps/backend/`, providing three core services: (1) an SSE streaming endpoint at `/api/sse/progress` for real-time AI progress broadcasting, (2) an AI proxy endpoint at `/api/ai/generate` that reads API keys from server env vars and reuses `@ac-canvas/ai-core` adapters, and (3) file upload/download endpoints at `/api/files/upload` and `/api/files/:fileId` using date-sharded local disk storage.

Purpose: Provide server-side AI execution and file storage, hiding client-side API keys (BKND-01, BKND-02). Enables production deployment where API keys remain server-side only.
Output: Working backend at localhost:3001 with all three service groups, tested and documented.

      - `removeClient(id)` — removes the client callback

      - Use streamSSE to create the stream
