    Step C: Create apps/web/e2e/ directory and playwright.config.ts
    Create file: `apps/web/e2e/playwright.config.ts`

    ```typescript
    import { defineConfig } from '@playwright/test'

    export default defineConfig({
      testDir: './specs',
      timeout: 30000,
      retries: 0,
      use: {
        headless: true,
        viewport: { width: 1280, height: 720 },
        baseURL: 'http://localhost:5173',
        // Disable animations to reduce flakiness
        actionTimeout: 10000,
      },
      // Only Chromium — per D-04
      projects: [
        {
          name: 'chromium',
          use: { browserName: 'chromium' },
        },
      ],
      // Do not run in CI — per D-08. No CI reporter needed.
      // Local-only: output HTML report for debugging
      reporter: [['html', { outputFolder: 'playwright-report', open: 'never' }]],
    })
    ```

    Key decisions:
    - `headless: true` — no GUI needed for automated E2E
    - `baseURL: 'http://localhost:5173'` — dev server URL (Vite 8 default)
    - `retries: 0` — local manual run, no flakiness retry needed
    - Only Chromium — per D-04
    - HTML reporter with `open: 'never'` — generate report file for inspection without auto-opening browser

    Step D: Add a `.gitkeep` to ensure the e2e directory is tracked

    Per D-05, the core flow is: create blank project -> add nodes -> connect -> Mock AI generate -> preview -> export PNG.

    ```typescript
    import { test, expect } from '@playwright/test'

    /**
     * Core E2E Flow: Create project -> Add nodes -> Connect -> Mock AI -> Preview -> Export PNG
     *
     * Prerequisite: Dev server running at http://localhost:5173
     * Run: pnpm --filter @ac-canvas/web test:e2e
     *
     * Per D-05: Only this single critical path is tested. No variant flows.
     * Per D-08: Local run only. No CI pipeline.
     */
    test.describe('Core creative flow', () => {
      test('create project, build node graph, trigger generation, preview, and export PNG', async ({ page }) => {
        // ── Step 1: Navigate to projects page and create a new project ──
        await page.goto('/projects')
        await page.waitForLoadState('networkidle')

        // Look for the "New Project" button and click it
        // The ProjectsPage has a "New Project" button (from Phase 7 implementation)
        const newProjectButton = page.getByRole('button', { name: /new project|新建/i })
        await expect(newProjectButton).toBeVisible({ timeout: 5000 })
        await newProjectButton.click()

        // After creating a project, the app navigates to / (canvas page)
        await page.waitForURL('/')
        await page.waitForLoadState('networkidle')

        // ── Step 2: Add nodes to the canvas ──
        // The node palette toolbar is at bottom-center of the canvas page.
        // It contains buttons for each node type: Prompt, Text-to-Image, Style, Merge, Preview.
        // Per the node editor implementation, these are labeled buttons in the toolbar.

        // Add a PromptNode
        const promptNodeBtn = page.getByRole('button', { name: /prompt|提示词/i })
        await expect(promptNodeBtn).toBeVisible({ timeout: 5000 })
        await promptNodeBtn.click()

        // Add a TextToImageNode
        const t2iNodeBtn = page.getByRole('button', { name: /text.to.image|文生图/i })
        await expect(t2iNodeBtn).toBeVisible()
        await t2iNodeBtn.click()

        // Add a PreviewNode  
        const previewNodeBtn = page.getByRole('button', { name: /preview|预览/i })
        await expect(previewNodeBtn).toBeVisible()
        await previewNodeBtn.click()

        // Verify three nodes were added (count node elements in the React Flow container)
        // React Flow renders nodes with data-testid or role-based selectors.
        // Node elements typically have class containing "react-flow__node"
        const nodeElements = page.locator('.react-flow__node')
        await expect(nodeElements).toHaveCount(3)

        // ── Step 3: Connect nodes (Prompt -> TextToImage -> Preview) ──
        // React Flow supports drag-connect: drag from source handle to target handle.
        // Source handles have class "react-flow__handle" with role "button" or data attributes.
        // We use the handle positioning (first node's output to second node's input).

        // Connect PromptNode (first) output to TextToImageNode (second) input
        // Source handle is typically the bottom/right handle of the node
        const handles = page.locator('.react-flow__handle')
        const handleCount = await handles.count()
        // Nodes have input (top) and output (bottom) handles. For 3 nodes,
        // we expect 6 handles (Promp:1out, T2I:1in+1out, Preview:1in = 3 nodes, 4 valid handles)
        // Actually: Prompt (1 out), T2I (1 in + 1 out), Preview (1 in) = 4 handles total
        // Handle 0 = Prompt output, Handle 1 = T2I input, Handle 2 = T2I output, Handle 3 = Preview input

        if (handleCount >= 4) {
          // Drag from Prompt output (handle 0) to T2I input (handle 1)
          await handles.nth(0).dragTo(handles.nth(1))
          // Drag from T2I output (handle 2) to Preview input (handle 3)
          await handles.nth(2).dragTo(handles.nth(3))
        }

        // Verify edges were created
        const edgeElements = page.locator('.react-flow__edge')
        await expect(edgeElements).toHaveCount(2)

        // ── Step 4: Configure node parameters and trigger Mock AI generation ──
        // Click the PromptNode to select it, then edit the prompt text in PropertyPanel
        const firstNode = nodeElements.first()
        await firstNode.click()

        // Find the prompt text input in the PropertyPanel (right sidebar or properties tab)
        // The PropertyPanel has text inputs for node parameters
        const promptInput = page.getByLabel(/prompt|提示词/i).or(page.getByPlaceholder(/enter prompt|输入提示词/i))
        if (await promptInput.isVisible()) {
          await promptInput.fill('A beautiful sunset over mountains')
        }

        // Trigger execution — either via auto-execute (debounced 180ms) or manual trigger
        // The execution should start automatically after node graph changes (debounced 180ms)
        // Wait for the execution to complete (nodes show "done" status)
        await page.waitForTimeout(1000) // Wait for auto-execute debounce + execution

        // ── Step 5: Preview the result ──
        // After execution, check that the PreviewNode shows a result
        // (MockAdapter returns a deterministic test image)
        // The execution status indicator shows progress
        const executingIndicator = page.locator('.executing-indicator, [data-testid="execution-status"]')
        // If visible, wait for it to disappear (execution complete)
        if (await executingIndicator.isVisible({ timeout: 2000 }).catch(() => false)) {
          await expect(executingIndicator).not.toBeVisible({ timeout: 15000 })
        }

        // Check that nodes show "done" status
        // Node status indicators are rendered as part of node components
        const doneIndicators = page.locator('.node-status-done, [data-status="done"]')
        await expect(doneIndicators).not.toHaveCount(0)

        // ── Step 6: Export as PNG ──
        // Click the export button in the top bar or toolbar
        const exportBtn = page.getByRole('button', { name: /export|导出/i })
        await expect(exportBtn).toBeVisible({ timeout: 5000 })
        await exportBtn.click()

        // Export dialog opens — verify it
        const exportDialog = page.getByRole('dialog')
        await expect(exportDialog).toBeVisible()

        // Set export format to PNG (default should be PNG per uiPreferencesStore defaults)
        // Click the export/confirm button in the dialog
        const confirmExportBtn = exportDialog.getByRole('button', { name: /export|导出|download|下载/i })
        await expect(confirmExportBtn).toBeVisible()
        await confirmExportBtn.click()

        // Verify export was triggered — dialog closes
        await expect(exportDialog).not.toBeVisible({ timeout: 5000 })

        // ── Verification: No console errors during the flow ──
        // Fail the test if there were console errors (unless they're known non-critical ones)
        const consoleErrors = page.locator('.console-error') // Not applicable — we check via listener
        // (Console error checking is done via page.on('console') in the test itself)
      })

      test('page loads without console errors', async ({ page }) => {
        const consoleErrors: string[] = []
        page.on('console', (msg) => {
          if (msg.type() === 'error') {
            consoleErrors.push(msg.text())
          }
        })

        await page.goto('/')
        await page.waitForLoadState('networkidle')

        // Filter out known non-critical errors (e.g., ResizeObserver loop)
        const criticalErrors = consoleErrors.filter(
          (e) => !e.includes('ResizeObserver') && !e.includes('favicon'),
        )
        expect(criticalErrors).toEqual([])
      })
    })
    ```

    Key testing patterns:
    - Uses Playwright's `test` and `expect` from `@playwright/test` (not vitest)
    - Routes: `/projects` for project creation, `/` for canvas view
    - Node interaction uses React Flow custom selectors (`.react-flow__node`, `.react-flow__handle`, `.react-flow__edge`)
    - Auto-execute is debounced at 180ms; wait `page.waitForTimeout(1000)` for safety
    - MockAdapter produces deterministic images without network calls (D-05: "Mock AI generate")
    - Export dialog is a `<dialog>` element rendered by ExportDialog component
    - Console error checking as secondary verification
    - All locators use fallback patterns (English/Chinese) for robustness

    Notes for the executor:
    - The test uses CSS class selectors that match React Flow's standard rendering classes. These are stable across React Flow v12.x versions.
    - Node status indicators: check how EngineStore renders node status in the actual UI. The class `.node-status-done` may need adjustment to match actual implementation.
    - Handle count logic: 3 nodes (Prompt: 1 out, TextToImage: 1 in + 1 out, Preview: 1 in) = 4 handles total. Verify this matches actual node handle configuration.
    - The test assumes auto-execute is configured (useAutoExecute hook active in CanvasPage). If the graph doesn't auto-execute, the test may need a manual execution trigger step.
