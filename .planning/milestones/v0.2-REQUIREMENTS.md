# Requirements: AI无限创意画布

**Defined:** 2026-06-29
**Validated:** 2026-07-02
**Core Value:** 用户可以在一个无限自由的画布上，通过拖拽节点的方式，精准可控地完成 AI 图文创作全流程——从创意构思、素材生成、逻辑编辑、效果调试到成品输出。

## v1 Requirements — COMPLETED

All 30 requirements validated. See traceability table for per-requirement outcomes.

### 无限画布 (Canvas)

- [x] **CANVAS-01**: 用户可以在无限延展的画布上自由绘制、拖拽、缩放和平移
  → **Validated** — Excalidraw v0.18 fork with pan/zoom/draw implements full infinite canvas
- [x] **CANVAS-02**: 画布支持 AI 生成图片元素的渲染和展示（AIElement）
  → **Validated** — AIElement custom Excalidraw element type renders generated images on canvas
- [x] **CANVAS-03**: 画布支持元素分层、分组、锁定、隐藏管理
  → **Validated** — LayerPanel provides reorder/lock/hide/group with visual feedback
- [x] **CANVAS-04**: 画布采用分片渲染优化，500+ 元素保持流畅操作
  → **Validated** — 2000x2000px chunk rendering, resolution-tiered rendering, 200MB LRU cache
- [x] **CANVAS-05**: 画布操作支持撤销/重做
  → **Validated** — HistoryStore captures canvas operations with keyboard undo/redo (Ctrl+Z/Y)
- [x] **CANVAS-06**: 项目可通过 IndexedDB 本地保存和加载
  → **Validated** — Dexie.js persistence, 180ms debounced auto-save, manual save/load

### 节点编辑器 (Node Editor)

- [x] **NODE-01**: 用户可通过 React Flow 集成实现节点拖拽和连线
  → **Validated** — @xyflow/react v12 integration with drag-and-drop and connection validation
- [x] **NODE-02**: 系统提供核心节点类型：PromptNode、TextToImageNode、StyleNode、MergeNode、PreviewNode
  → **Validated** — 5 node types with 30+ socket types
- [x] **NODE-03**: 节点执行引擎支持拓扑排序、同层并行执行、增量执行（Dirty标记）
  → **Validated** — toExecutionLayers topological sort, findAffectedDownstream dirty-path
- [x] **NODE-04**: 用户可在右侧参数面板调节节点参数
  → **Validated** — PropertyPanel with shadcn/ui form controls per node schema
- [x] **NODE-05**: 节点图支持序列化/反序列化（保存/加载）
  → **Validated** — Graph state serialized alongside canvas, unified save/load
- [x] **NODE-06**: 节点操作支持撤销/重做
  → **Validated** — HistoryStore captures node graph operations alongside canvas
- [x] **NODE-07**: 节点支持子分组（Group），防止"节点汤"
  → **Validated** — GroupNode component with collapse/expand, named groups

### AI 核心 (AI Core)

- [x] **AI-01**: 系统通过 OpenAI DALL-E 3 适配器实现文生图
  → **Validated** — OpenAiAdapter implements text-to-image with quality/style/error handling
- [x] **AI-02**: 系统通过 Stability.ai 适配器实现文生图和图生图
  → **Validated** — StabilityAdapter implements text-to-image and image-to-image
- [x] **AI-03**: 系统提供 MockAdapter 用于离线调试
  → **Validated** — MockAdapter returns SVG test images, zero API calls, instant response
- [x] **AI-04**: AI 请求队列 + 速率限制保证调用稳定性和可控性
  → **Validated** — AIQueueStore with per-provider FIFO queues, sliding-window rate limiter
- [x] **AI-05**: 用户可自带 API Key 和自定义 API URL（BYOK 模式）
  → **Validated** — ProviderStore with AES-256-GCM encryption, Dexie persistence
- [x] **AI-06**: 系统提供 Prompt 构建器和模板系统
  → **Validated** — Tempura template engine, 7 templates across providers, 4-source context merge
- [x] **AI-07**: 生成进度通过 SSE（Hono streamSSE）实时推送到前端
  → **Validated** — Hono SSE broadcast + SSEService + useSSEProgress hook
- [x] **AI-08**: 节点引擎与 AI 适配器完成桥接对接
  → **Validated** — createAiExecutor bridge wiring registry→provider→prompt→progress→blob

### UI 界面 (UI)

- [x] **UI-01**: 提供工具栏、侧边栏、素材面板等基础 UI 组件
  → **Validated** — AppShell with toolbar, TabbedSidebar, PropertyPanel, dark mode
- [x] **UI-02**: 用户可将画布导出为 PNG/JPG 格式

### 后端服务 (Backend)

- [x] **BKND-01**: 后端提供 AI 代理 API，隐藏客户端 API Key
  → **Validated** — Hono AI proxy routes, dual-mode (direct/proxy) dispatch
- [x] **BKND-02**: 后端支持文件上传和下载服务
  → **Validated** — File upload/download with better-sqlite3 metadata, flat file storage

### 测试 (Testing)

- [x] **TEST-01**: 节点引擎核心逻辑单元测试覆盖
  → **Validated** — 418 tests across 38 files including engine, stores, hooks, resolvers
- [x] **TEST-02**: AI 适配器 Mock 测试
  → **Validated** — MockAdapter tests, adapter interface conformance tests
- [x] **TEST-03**: 核心 E2E 流程测试（创建项目→拖拽节点→AI生成→导出）
  → **Validated** — 3 Playwright tests pass in 14.2s (core flow verified)

## Out of Scope

| Feature | Reason |
|---------|--------|
| AI 文生视频 / 图生视频 | 高 API 成本且复杂度大，推迟至 v0.2+ |
| 云端实时同步协作 | 本地单机优先，协作功能在 v0.3 |
| 模板广场 | 生态功能，v0.3 考虑 |
| Electron 桌面应用 | v0.3 阶段考虑 |
| 移动端适配 | v1.0 阶段 |
| 3D 节点创作 | 长期演进 |
| 自训练 AI 模型 | 使用现有商业模型 API |
| WebSocket 实时通信 | SSE 已满足需求，过度设计 |
| 计费/订阅系统 | v0.1 不需商业化 |
| Replicate 适配器 | MVP 保留 OpenAI + Stability 两条主流即可 |
